Class threadpool

boost::object::threadpool

Synopsis

// In header: <boost/object/threadpool.hpp>


class threadpool : public boost::object::object {
public:
  enum PickBehavior;
  // construct/copy/destruct
  threadpool(int, object * = 0);
  threadpool(int, PickBehavior = PickLeastUsed, object * = 0);

  // protected member functions
  auto_slot thread * getThread();
  auto_slot void setPickBehavior(PickBehavior);
};

Description

The threadpool manages multiple threads. Either use getThread(), or simply make another bobject the child of this pool to get a thread according to your PickBehavior.

Notes for when you compile with BOOST_OBJECT_DISABLE_HIERARCHY: 

All bobjects assigned to a thread of this pool need to be directly deleted ( using delete or they need to go out of scope, BObject::deleteLater() is not an option ) before threadpool::~threadpool() is invoked. Otherwise it will cause undefined behavior!
