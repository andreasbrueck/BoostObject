Description

Template Parameters

  1. typename... FUNCTION_SIGNATURE

    The signature of the signal. Can be a function signature or a list of parameters, the first being the return type:

    • signal_t<void(int)>

    • signal_t<void> Evaluates to signal_t<void()>

    • signal_t<void,int> Evaluates to signal_t<void(int)>

    • signal_t<> Evaluates to signal_t<void()>

signal_t public member functions

  1. template<typename SLOT_TYPE, class... DEFAULT_ARGS> 
  boost::object::connection connect(SLOT_TYPE & slot, DEFAULT_ARGS && ...);
    Connects slot to this signal.

    slot may be one of the following:

    • A bobject slot

    • A bobject signal

    • A functor/lambda

    • A (static) function

    When connecting with a functor/lambda/(static)function, the connection will stay alive for as long as the signal does.

    When connecting with another signal or a slot, the connection will automatically disconnect if either the owner bobject of slot is destroyed, or this signal - whichever happens first.

    In both cases, you can also disconnect the connection by calling disconnect() on the returned boost::object::connection instance.

    Parameters:

    slot

    The destination you want to connect this signal to
